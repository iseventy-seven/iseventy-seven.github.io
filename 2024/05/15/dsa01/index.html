02-A2 抽象数据类型：从数组到向量

02.Vector.A2.ADT.From_Array_To_Vector

  • 数组

    • 一段连续的内存空间，被均匀的划分为若干个单元，每一个单元都与 之间的某一个整数编号相互对应。
    • 每个元素均由（非负编号）唯一指代，并可直接访问其物理地址，故亦称作线性数组（Linear Array）。对于数组元素 A[i]，其物理地址为 A + i * s，s 为单个元素占用的空间量。

  • 向量

    • 是数组的抽象与泛化，由一组元素按线性次序封装而成，各元素与 内的秩（Rank）一一对应，访问方式称为循秩访问（Call By Rank）。
    • 元素的类型不限于基本类型，便于操作、管理，更加简化、统一、安全。

  • 向量 ADT 接口

02-B2 可扩充向量：分摊

02.Vector.B2.Extendable_Vector.Amortization

  • 平均复杂度或期望复杂度（Average/Expected Complexity）
    • 根据数据结构各种操作出现概率的分布，将对应的成本加权平均。
    • 由于对于各种可能的操作，都作为独立事件分别考察，割裂了操作之间的相关性和连贯性，因此不能准确地评判数据结构和算法的真实性能。
  • 分摊复杂度（Amortized Complexity）
    • 对数据结构连续地实施足够多次操作，所需总体成本分摊至单次操作。
    • 从实际可行的角度出发，对一系列操作做了整体的考量，更加忠实地刻画了可能出现的操作序列，可以更为精确地评判数据结构和算法的真实性能。

  • 唯一化：正确性

    • 不变性：在当前元素 V[i] 的前缀 V[0, i) 中，各元素彼此互异。
      • 利用数学归纳法
      • 当 i = 1 时，前缀只有一个元素，成立；
      • 当 i = n 时，假设条件成立，即 V[0, n) 中，各元素互异；
      • 则当 i = n + 1 时，对于第 i 步操作，有两种情况
        • 若 V[n] 在前缀中的查找失败，即前缀中不含 V[n]，则将其插入，此时 V[i] 的前缀 V[0, n + 1) 中，各元素互异。
        • 若 V[n] 在前缀中的查找成功，即前缀中包含 V[n]，则将其删除，此时 V[i] 由 V[n + 1] 变为 V[n]，其前缀仍是 V[0, n)，上述假设可知各元素互异。
      • 综上，即证。
    • 单调性：随着反复的 while 迭代
      • 当前元素前缀的长度单调非降，且迟早增至 _size
      • 当前元素后缀的长度单调下降，且迟早减至 0
      • 综上，算法必然终止，且至多迭代 轮。

    • 二路归并（2-Way Merge）：将两个有序序列合并为一个有序序列，即 。

  • 二路归并：实现

    勘误 视频中上述写法在 for (Rank i = 0; i < lb; B[i] = A[i++]); 一句有问题，该语句会先取出 A[i] 的值，再 i++，之后赋值，因此实则是将 A 的第 i 项赋给了 B 的第 i + 1 项，最终导致 A 中的所有项均为垃圾值。可以按照讲义，写作 for (Rank i = 0; i < lb; i++) {B[i] = A[i]}，也可以写作 for (Rank i = 0; i < lb; B[i++] = A[i]);，即先取出 A[i]，赋值给 B[i]，再 i++。

    执行上述代码时，除了结果均为垃圾值，还会卡在 delete [] B; 一句，这是因为在之前的 for 语句中，为 B 的赋值越界了，原本 T* B = new T[lb];，但是会执行 B[lb] = A[lb - 1];，此时 B 不能正常 delete。

    • 在 if 判断中，j < lb 表示对于前子向量，还没有越界，lc <= k || (B[j] <= C[k]) 表示当后子向量越界或前子向量的当前元素不大于后子向量的当前元素时，将前子向量的当前元素取出，放入到结果向量中。这里应用了 c++ “短路求值”的语法特性，当后子向量越界时，lc <= k 为真，不会执行后半段语句，防止了运行错误。这里也可以理解为，在 k = lc 的位置假想了一个哨兵，其值为无穷大，因此当越界时，B[j] 一定小于 C[k]，此时两个条件均为真，可以将 B[j] 取出。对于后子向量的相关判断同理。

    在两条 if 判断中，对于前子向量判断 B[j] <= C[k]，而对于后子向量判断 C[k] < B[j]，这里不取等是为了保证算法的稳定性。例如前后子向量分别为 {1a, 1b, 4a}、{1c, 3, 4b}，若都取 <=，得到结果为 {1a, 1c, 1b, 3, 4a, 4b}，而按照上述取法，得到结果为 {1a, 1b, 1c, 4a, 4b}。
